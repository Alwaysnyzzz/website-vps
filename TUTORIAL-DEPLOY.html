Persiapan & Koneksi SSH
Hal-hal yang dibutuhkan sebelum mulai
Yang Kamu Butuhkan
  • ✓ VPS dengan OS Ubuntu 22.04 LTS (minimal 1 CPU, 512MB RAM)
  • ✓ IP publik VPS (contoh: 123.456.789.0)
  • ✓ Akses root atau user dengan sudo
  • ✓ Akun Supabase (gratis di supabase.com)
  • ✓ Akun Pakasir (app.pakasir.com) dengan project aktif
  • ✓ Terminal / CMD / PowerShell di komputer kamu
Koneksi SSH ke VPS
bash — di komputer kamu
# Format: ssh username@IP_VPS
ssh root@123.456.789.0

# Kalau pakai file key (.pem)
ssh -i /path/to/key.pem root@123.456.789.0

# Jika muncul "Are you sure?" ketik:
yes
Update sistem dulu
bash — di VPS
apt update && apt upgrade -y
apt install -y curl wget git unzip nano ufw
⚠️
Semua perintah di bawah ini dijalankan di dalam VPS (setelah SSH), kecuali ditulis "di komputer kamu".

Install Nginx & Node.js
Web server dan runtime JavaScript
Install Nginx
bash
apt install -y nginx

# Aktifkan nginx otomatis saat boot
systemctl enable nginx
systemctl start nginx

# Cek sudah jalan
systemctl status nginx
# Harus ada: ● nginx.service — Active: active (running)
Install Node.js 20 (via NodeSource)
bash
# Tambah repo Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -

apt install -y nodejs

# Verifikasi versi
node -v    # harus: v20.x.x
npm  -v    # harus: 10.x.x
Install PM2 (Process Manager)
bash
npm install -g pm2

# Verifikasi
pm2 -v
Setup Firewall (UFW)
bash
ufw allow OpenSSH
ufw allow 'Nginx Full'
ufw --force enable

# Cek status
ufw status

Upload File Frontend
Salin semua file HTML/CSS/JS ke VPS
Buat folder dan set permission
bash — di VPS
# Buat folder web root
mkdir -p /var/www/html

# Set ownership ke current user
chown -R $USER:$USER /var/www/html
chmod -R 755 /var/www/html
Upload file dari komputer kamu (pilih salah satu cara)

Cara A — SCP (Terminal)

bash — di komputer kamu
# Upload semua file frontend sekaligus
scp index.html login.html register.html \
    isisaldo.html isisaldo-pay.html pay.html \
    style.css style-extra.css loading.css \
    config.js auth.js script.js \
    root@123.456.789.0:/var/www/html/

# Upload folder image (kalau ada)
scp -r image/ root@123.456.789.0:/var/www/html/image/

Cara B — Unzip langsung di VPS

bash
# Upload zip dulu via scp
scp vps-web-updated.zip root@123.456.789.0:/root/

# Di VPS: extract ke web root
cd /root
unzip vps-web-updated.zip
cp -r vps-output/. /var/www/html/
# JANGAN copy server.js, package.json ke /var/www/html!

5
Konfigurasi Supabase
Buat tabel transactions dan ambil service key
Buat tabel transactions di Supabase
  1. Buka supabase.com → login → pilih project kamu
  2. Klik menu SQL Editor di sidebar kiri
  3. Klik New query
  4. Copy-paste isi file supabase-migration.sql ke editor
  5. Klik tombol RUN
✅
Jika berhasil, akan muncul "Success. No rows returned" di bagian bawah.
Ambil Service Role Key
  1. Di Supabase, klik Settings (ikon gear)
  2. Pilih API
  3. Di bagian Project API keys, temukan service_role
  4. Klik Reveal → Copy key-nya
  5. Paste ke SUPABASE_SERVICE= di file .env tadi
🔐
Service Role Key sangat sensitif! Jangan pernah di-commit ke Git atau ditaruh di file frontend yang bisa dibaca publik.

Konfigurasi Nginx
Setup web server dan proxy ke backend
Copy dan terapkan nginx.conf
bash
# Backup config lama
cp /etc/nginx/sites-available/default /etc/nginx/sites-available/default.bak

# Copy config baru dari zip kita
cp /root/vps-output/nginx.conf /etc/nginx/sites-available/default

# Test syntax nginx (PENTING!)
nginx -t
# Harus muncul: syntax is ok + test is successful

# Reload nginx
systemctl reload nginx
Isi nginx.conf (untuk referensi)
nginx
server {
    listen 80;
    server_name _;  # ganti dengan domain jika ada

    root /var/www/html;
    index index.html;

    # Semua /api/* → backend Node.js port 3000
    location /api/ {
        proxy_pass         http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_read_timeout 30s;
    }

    # Form isi saldo
    location = /isisaldo {
        try_files /isisaldo.html =404;
    }

    # Halaman bayar QRIS (/isisaldo/ORDER-ID)
    location ~ ^/isisaldo/(.+)$ {
        try_files /isisaldo-pay.html =404;
    }

    location = /login    { try_files /login.html =404; }
    location = /register { try_files /register.html =404; }
    location = /product  { try_files /product.html =404; }

    location / { try_files $uri $uri/ /index.html; }

    gzip on;
    gzip_types text/plain text/css application/javascript;
}
7
Jalankan Backend dengan PM2
Agar backend tetap jalan walau terminal ditutup atau VPS restart
Start backend
bash
cd /var/www/backend

# Start dengan PM2
pm2 start server.js --name dzzxnzz-backend

# Lihat status
pm2 status
# Harus: dzzxnzz-backend | online

# Lihat log real-time
pm2 logs dzzxnzz-backend
Auto-start saat VPS reboot
bash
# Generate startup script
pm2 startup
# Akan muncul perintah, COPY dan jalankan perintah itu
# Contoh: sudo env PATH=... pm2 startup systemd -u root --hp /root

# Simpan daftar proses PM2 saat ini
pm2 save
ℹ️
Setelah pm2 startup, akan ada 1 baris perintah panjang yang muncul. Copy dan jalankan baris itu agar PM2 otomatis start saat VPS restart.
Perintah PM2 yang sering dipakai
PerintahFungsi
pm2 statusLihat semua proses yang berjalan
pm2 logsLihat log semua proses
pm2 logs dzzxnzz-backendLog backend saja
pm2 restart dzzxnzz-backendRestart backend
pm2 stop dzzxnzz-backendStop backend
pm2 delete dzzxnzz-backendHapus dari PM2

Setup Webhook Pakasir
Agar Pakasir otomatis notif ke server saat bayar berhasil
URL Webhook kamu
url
http://123.456.789.0/api/webhook/pakasir

Ganti 123.456.789.0 dengan IP VPS kamu.

Cara setup di Pakasir Dashboard
  1. Login ke app.pakasir.com
  2. Masuk ke project toko-digital
  3. Buka menu Settings atau Webhook
  4. Set Webhook URL: http://IP_VPS/api/webhook/pakasir
  5. Set header secret:
    Key: x-webhook-secret
    Value: dzzxnzz-webhook-secret-2026
  6. Simpan perubahan
💡
Nilai WEBHOOK_SECRET di file .env harus sama persis dengan yang kamu set di Pakasir Dashboard.

Troubleshooting
Solusi masalah yang paling sering muncul
502 Bad Gateway
bash
# Backend tidak jalan → cek PM2
pm2 status
pm2 logs dzzxnzz-backend

# Kalau error "MODULE_NOT_FOUND"
cd /var/www/backend && npm install

# Restart backend
pm2 restart dzzxnzz-backend
QR tidak muncul / error saat top up
bash
# Cek log backend
pm2 logs dzzxnzz-backend --lines 50

# Kemungkinan penyebab:
# 1. PAKASIR_API_KEY salah → cek di .env
# 2. PAKASIR_PROJECT salah → cek nama project di pakasir.com
# 3. .env tidak terbaca → pastikan ada di /var/www/backend/.env
Coins tidak bertambah setelah bayar
bash
# Cek log webhook
pm2 logs dzzxnzz-backend | grep WEBHOOK

# Kemungkinan penyebab:
# 1. SUPABASE_SERVICE kosong → isi di .env lalu restart
# 2. Webhook URL salah di Pakasir
# 3. WEBHOOK_SECRET tidak cocok

# Setelah edit .env, WAJIB restart:
pm2 restart dzzxnzz-backend
Halaman 404 Not Found
bash
# Cek file ada di web root
ls /var/www/html/

# Cek nginx config benar
nginx -t

# Lihat error log nginx
cat /var/log/nginx/error.log | tail -20
Cara update file setelah ada perubahan
bash
# Update file frontend (HTML/CSS/JS)
scp namafile.html root@IP_VPS:/var/www/html/
# Tidak perlu restart apapun

# Update server.js backend
scp server.js root@IP_VPS:/var/www/backend/
pm2 restart dzzxnzz-backend
